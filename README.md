# Disciplina de Back-End

Este repositório contém material e exemplos relacionados à disciplina de desenvolvimento back-end utilizando Node.js e tecnologias associadas.

## Conteúdo da Disciplina

1. **Node.js**
   - JavaScript Runtime baseado no motor V8 do Chrome.
   - Executa JavaScript fora do navegador.

2. **JavaScript Runtime**
   - Conceito de ambiente de execução para scripts JS.
   - Event Loop e modelo de concorrência.

3. **Chrome's V8 JavaScript engine**
   - Compilação Just-In-Time (JIT).
   - Gerenciamento de memória e garbage collector.

4. **Assíncrono orientado a eventos**
   - Callbacks, Promises e async/await.
   - Non-blocking I/O e o loop de eventos.

5. **Iniciando um servidor web**
   - Criar servidor HTTP básico.
   - Escutar portas e tratar requisições.

6. **O módulo http**
   - API nativa para criar servidores.
   - Objetos `request` e `response`.

7. **Request e response**
   - Estrutura das requisições HTTP.
   - Métodos, headers e body.
   - Como enviar respostas ao cliente.

8. **npm**
   - Gerenciador de pacotes do Node.js.
   - Comandos `init`, `install`, `run`.

9. **Instalar e usar módulos**
   - Dependências locais e globais.
   - Criar módulos próprios e importá-los com `require`/`import`.

10. **ECMAScript 2015 - ES6**
    - Novas sintaxes e funcionalidades.

    - **Orientação a Objetos**
      - Classes, herança e módulos.
    - **Novidades em relação ao ES5**
      - Arrow functions, `let`/`const`, template literals, destructuring, promises, entre outros.

11. **Web server com Express**
    - Framework minimalista para servidores.
    - Roteamento, middleware e estrutura MVC.

12. **Configurando o servidor com Express**
    - Instalando o Express via npm.
    - Criar instância `express()` e definir rotas.

13. **Estrutura da aplicação**
    - Organização de pastas e arquivos.

14. **App.js**
    - Arquivo principal que inicia a aplicação.
    - Importa dependências, configura middleware e define rotas.

15. **Pasta `bin`**
    - Scripts executáveis (por exemplo, `www` para iniciar o servidor).

16. **Pasta `public`**
    - Arquivos estáticos (CSS, JS, imagens).

17. **Pasta `routes`**
    - Definições de rotas separadas por recurso.

18. **Pasta `views`**
    - Templates de visualização (EJS, Pug, Handlebars, etc.).

---

Este README serve como guia básico para os tópicos cobertos na disciplina de back-end. Exemplos de código e exercícios podem ser adicionados ao repositório para complementar o aprendizado.
